## 项目规则

- 所有文件管理操作继续遵守 [Worker Invocation 保护规则](./PROJECT_RULES.md#worker-invocation-保护)。
- 新增批量、目录、跨存储、ZIP、统计、索引等功能时，必须默认按 Worker invocation 限制设计，不得在单次请求内同步处理不可预估数量的文件。

## 批量移动 / 大目录移动

- 将批量移动实现为后台任务，而不是同步 `rename` / `copy + delete`。
- 大目录移动必须拆为阶段执行：
  - 复制源目录到目标目录。
  - 校验目标目录对象数量与关键路径是否完整。
  - 复制确认完成后再删除源目录。
- S3/R2 同挂载目录移动当前依赖 `copyDirectoryRecursive()` 后 `deleteDirectoryRecursive()`，后续应避免在复制部分成功时删除源目录。
- WebDAV MOVE 当前也是 `copyItem()` 后 `batchRemoveItems()`，后续应在复制结果为 `partial/failed` 时阻止删除源。
- 超大目录移动应保存分页游标和阶段状态，支持 Workflow 分批续跑。

## 网盘管理完善优先级

### P0：Worker Invocation 保护与任务可靠性

- 所有新增文件管理能力必须先判断是否存在不可预估数量的文件操作；存在时必须走后台任务、分片执行、断点续跑。
- 批量复制、批量移动、批量删除、目录重命名、目录统计、批量 ZIP 都需要保持 invocation 异常保护。
- 任务进度需要区分“路径总数”和“递归文件总数”，避免 100% 但任务仍在目录内部继续处理。
- 任务状态展示需要明确阶段：预扫描、复制中、删除清理中、打包中、统计中、完成、部分失败、失败。
- Worker 触发 invocation / subrequest / timeout 风险时，任务应保存当前游标并降级单批数量，下一轮继续处理。

### P1：目录属性统计

- 文件属性面板补齐目录大小、目录文件数量、子目录数量、最近修改时间汇总。
- 目录统计必须做成异步统计任务，不允许打开属性面板时同步递归扫描大目录。
- 统计结果需要缓存，文件变更、移动、删除、上传后按路径失效或标记脏数据。
- 属性面板显示统计状态：未统计、统计中、已统计、统计失败、数据可能过期。

### P1：回收站 / 软删除

- 批量删除优先支持软删除到回收站，降低误删风险。
- 回收站支持恢复、永久删除、按时间自动清理。
- 删除任务需要记录原路径、删除时间、操作者、对象类型和大小摘要。
- 大目录软删除仍需要后台任务分片执行，不能同步移动整个目录树。

### P2：复制 / 移动冲突策略

- 复制、移动、目录重命名时补齐冲突策略：跳过、覆盖、自动重命名、保留较新。
- 冲突策略需要在任务创建前明确传入后端，任务详情中展示每个文件的处理结果。
- 跨存储复制时默认使用安全策略：目标存在则跳过或自动重命名，避免无提示覆盖。

### P2：索引化搜索与目录缓存

- 建立文件索引，用于搜索、目录大小统计、文件数量统计和快速刷新。
- 索引更新走后台增量任务，避免每次刷新都完整拉取远端目录。
- 支持目录脏标记，上传、删除、移动、重命名后只刷新受影响路径。
- 刷新按钮继续保持后台刷新体验，列表旧数据不应被清空成长期加载态。

### P3：大目录 ZIP 下载

- 批量 ZIP 下载先支持中小规模文件篮场景。
- 大目录 ZIP 需要后端清单任务：先递归展开清单，再分片读取并生成下载产物。
- ZIP 产物可优先落到 R2/S3 临时目录，再返回临时下载链接。
- 需要设置最大文件数、最大总大小、单文件失败策略和临时文件过期清理。

### P3：预览与编辑增强

- 文本编辑继续限制大文件，默认大于 1MB 不进入预览和编辑。
- 后续增强 Markdown 渲染、JSON 格式化、代码高亮、PDF 预览。
- 可编辑后缀继续保留后台配置，默认列表只作为初始值，不写死到界面逻辑中。

### P3：权限与审计

- 增加文件操作日志：上传、删除、恢复、移动、重命名、编辑保存、分享链接生成。
- 分享链接支持过期时间、访问次数限制、密码和撤销。
- 多用户场景下补齐目录级只读、可写、可删除权限。
